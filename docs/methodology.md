# Methodology

## 1. Data Understanding
The notebook begins by importing the transaction dataset, checking the structure, reviewing columns, and validating missing values and duplicates.

## 2. Exploratory Data Analysis
The analysis reviews high-value transactions, merchant behaviour, monthly and yearly trends, transaction type patterns and top-client concentration.

## 3. Risk-Focused Analysis
Clients are grouped using transaction behaviour indicators such as online transaction ratio and transaction amount patterns. These indicators help highlight clients or merchants that may require deeper investigation.

## 4. Business Interpretation
The project does not claim to be a deployed fraud detection model. It is an analyst-style EDA project designed to support early fraud monitoring and investigation prioritisation.
